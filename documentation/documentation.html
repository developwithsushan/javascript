Getting Started

What is JavaScript ?

JavaScript is a dynamic, weakly typed programming language which is complied at runtime. It can be executed as a part of a web page in a browser or directly on any machine ("host environment").
JavaScript was created to make webpages more dynamic (eg change content on a page directly from inside the browser).
Originally it was called LiveScript but due to the popularity of Java, it was renamed to JavaScript.
JavaScript is totally independent form Java and has nothing in common with Java.

A Brief introduction to JavaScript

We can define JavaScript in many different ways.

JavaScript is a high level, object oriented, multi-paradigm programming language.

  • A programming language is basically just a tool that allows us to write code that will instruct a computer to do something.
  • Javascript is a high level language which means that we managing the computer's memory while it runs our program. In JavaScript there are a lot of so called abstractions over all these small details, that we dont need to worry about and this makes the language a lot easier to write and to learn.
  • JavaScript is object oriented all that means is that the language is mostly based on concepts of objects for storing most kinds of data.
  • JavaScript is also a multi-paradigm language - meaning that its so flexible and versatile that we can use all kinds of different programming styles such as imperative and declartive programming.

JavaScript in Action (How is Javascript executed ?)

Lets say we write our JavaScript code and want it to have some effect on the webpage. If we talk about the browser as the environment where we run our script. Then we have one important thing built into any environment where you want to run JavaScript code and that's JavaScript Engine (it's built into the browser, in chrome, its v8 in firefox the name is spider monkery and of course other browsers also either reuse these engines or have their own engines.)
Now the job of the engine is to parse code (read and understand your JavaScript code) then on the fly complie it to machine code because machine code executes faster and then executes that machine code. This all happens in the broswer with the help of the JavaScript Engine and then when the code is executed, we have that effect on our webpage.

NOTE: The browser has a built in tool that takes our code, compiles it, optimizes it and executes it (all of that happens on a single thread).

Dyanamic? Weakly Typed

JavaScript is a dyanamic, interpreted programming language but it's also a weakly typed programming language

Dynamic, interpreted programming langauge
The dynamic interpreted part means that it is not pre-compiled, other programming languages like C++ are compiled during or after development, so before we share them with the end users, JS is on the fly compiled and that means that the code is evaluated and executed at runtime. It also means that the code can change at runtime (eg type of a variable). Dynamic thing about JavaScript means that it is parsed and interpreted and compiled at runtime and that therefore it is able to do certain things, for example, switching the data of a variable other programming language are not allowed to do.

Weakly Typed programming langauge
Data type are assumed (eg assigned to variables) automatically. You dont define that some variable has to hold a certain value (eg, a number).
Data types are not set in stone but can change.

JavaScript runs on a host environment

JavaScript runs on a host environment, that means JavaScript engine can be part or can be executed in different environments. The most well known environment is the browser, modern browsers have JavaScript engine built in and they are capable of executing javascript code but we can also run Javascript in other environments, for example on the server side, so right on a computer without having a broswer in between.

Browser Side Other (e.g., Server Side)
Javascript was invented to create more dynamic websites by executing in the browser. Google's JavaScript Engine (v8) was extracted to run JS anywhere(called Node JS).
Javascript can manipulate the HTML code, css, send background HTTP requests and much more. Node JS can be executed on any machine and is therefore often used to built web backends.
Javascript can't access the local filesystem, interact with operating system etc. Node JS can access the local filesystem, interact with the OS etc. It cant manipulate HTML or CSS.

Introducing Variables and Constants

Variables & Constants : That's a core feature not just of JavaScript but of most programming languages.
A Variable is a data container or data storage where the value can change, doesn't have to but often it will.
In JavaScript we create a variable by using the let keywords, then name of your choice with some naming rules, then an equal sign and then the value you want to store in the variable. 

let username = 'Sushan';

Now with a variable created you can also always reassign it and store a new value in there. 

let username = 'Sush';

Constants where the value most not change.
JavaScript is a case sensitive.
